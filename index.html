Stage 0: Getting Started with HTML

Introduction to the Web

World Wide Web

The web is a collection of html documents that are all linked together by Hyper Links.

HTML

HTML = Hyper Text Markup

Everything online is made up of HTML.

HTML is made up of:

  • Text content (what you see)
  • Markup (what it looks like)
  • References to other documents (images, videos)
  • Links

HTML Mark-up

HTML consists of opening tags and closing tags, which make up an element.

HTML tags:
  • "em"= italics
  • "b"= bold
  • "p" = paragraph
  • "h1" to "h6" = Defines HTML headings
  • "div" = defines a section in a document
  • "!DOCTYPE" = Defines the document type
  • "br" = Defines a single line break
  • "img" = Defines an image
  • "a"= anchor (this is a link)

HTML Attributes

HTML attributes are listed after the elements, but before the corresponding value. Links (a element) and images (img element) are self-closing and are considered Void tags beacuse they do not contain any content.

More information...

HTML Whitespaces

Turns everything into one line. If we want to seperate the contents into multiple lines, we write "br" which is a Void tag for "break". The "p" tag is also used and is not a Void tag because it includes content.

Inline and Block Elements

These two types of elements are used to separate content. An example of an inline element is "br" because it adds a break in the content and places the text on the next line below. An example of a block element is "p" because it creates an invisible blox around the content and also has a height and a width. Span is also an inline element and Div is a block element.

Page Structure

Websites are created like a house. The HTML is the structure, the CSS is the styling or decor, and Javascript is the functional parts or garage door.

The Basics of the Web and HTML

HTML, CSS, and DOM

HTML is the language of the web and includes syntax + rules.

CSS is a language that uses syntax + rules to change the way an element looks on the page.

Tags are part of the lement that make up an HTML tree.

DOM ( Document Object Model) is what puts everything together. The tree that we see in developer tools is the DOM tree that the browser built from the HTML document.

Each tag has an element that describes what type it is. An element consists of a beginning tag, a closing tag, and everything in between.

Boxes, Boxes, Boxes Everywhere

Even if the shape is a circle, in HTML everything is made up of boxes. If you want to right-click to inspect an element, you will see an assortment of diffrent boxes.

Understanding CSS (aka Cascading Style Sheet)

Now for some general laws...just like in our society; we have national laws, state laws, local municipality laws, and house rules. CSS has laws/rules. The most specific ruleis applied to every element. Thus, our "house laws" would be the same as our "national laws".

Cascading means that rules are applied not only to the elements they directly match but also to all of the elements' child elements. However, if a child element has multiple overlapping rules defined for it, the more specific rules apply.

When using CSS it can be helpful to think about it as a Search-n-Replace tool. You identify a "class" or "tag" of the element you want to match and then what you want to do with, or what property values to replace with different ones. Also, where and in what order matter. Styles can be defined in different places and are applied in the following order, with definitions further down `the list overwriting previous definitions:

  • Default style of a browser (different browsers have slightly different styles)
  • Stylesheet in a separate file (mostly used)
  • Stylesheet inside a HTML (used for smaller projects, not ideal)
  • Inline style in an element (should be avoided)

Inheritance: relies on the ancestor/descendant relationship to operate. The idea by which properties are applied, not only to a specified element, but also it's descendants. It relies on the document tree, the hierarchy of HTML elements. Child elements may inherit CSS properties from any ancestor element enclosing it; they can also inherit text related properties, but box related properties are not inherited.

Properties that can be inherited:

  • color
  • font
  • letter spacing
  • line height
  • list style
  • text align/indent
  • text transform
  • visibilty
  • white space
  • word spacing

Properties that can't be inherited:

  • background
  • border
  • display
  • float & clear
  • height & width
  • margin
  • min & max/ height & width
  • outline
  • overflow
  • padding
  • position
  • text decoration
  • vertical align
  • z index

Selectors

Selectors are what allows you to targe specifc HTML elements and apply style to them. By doing this, you can condense your CSS to avoid unnecessary repetition, which waste time, and time is precious. It is recommended that CSS and HTML are kept in two different files, by keeping the design away from the document they are easier to edit. Selectors can be combined. Below is a list of different type of Selectors.

ID Selectors
Most powerful type of selector, beat out other types of selectors and styles. Having lower specificity selectors are easier to override.

Class Selectors
Most useful and versatile. Well supported in all browsers. Add multiple classes separated by space on HTML elements. Also JavaScript stuff you can do specifically for manipulating classes.

Tag Selectors
Most useful when changing properties that are unique to HTML element. Like setting the list style on a "ul" or tab-size on a "pre". Typically more useful to have class define styling that you can use on any HTML element.

Attribute Selector
More useful than classes because they have the same specificity value, but they can be any attribute, not just "class". They also can have a value you can select by.

Positional Selectors
Several positional selectors beyond: nth-child; using simple expressions (like 3n="every third") you can select elements based on their position in the HTML. Other Pseudo Selectors ":empty" is one of many pseudo selectors, which you can recognize by the ":". They typically represent something you couldn't know just by the element and attributes alone. *These are different than pseudo elements, which are recognizable by "::"-they are responsible for adding things to the page by the things they select.

Revisiting the BOX

We know that everything on a webpage is made up of a box. Which consists of 4 parts:

  1. Content- this is the image or text that appears on the website, comparable to the nucleus of the box, the other layers serve to protect it so other things don't get to close or overlap
  2. Padding- this clears an area around the content, is affected by background color, and is the space that protects the content directly
  3. Border- inherits color property of box, incases the padding and content
  4. Margin- goes around border, outer most layer, provides space between boxes, transparent

To deal with sizing issues, use: "box-sizing:border-box...however...to make sure older browsers are able to display this CSS, you must add browser specific prefixes "webkit, moz, ms". If we set box size to percentage rather than pixels, the box sizing will adjust to size of screen. When positioning boxes remember: *divs are block elements so by default they take up the entire width of a page, to deal with this we can add the "display:flex" to appropriate CSS will override this behavior and let divs appear next to each other!* See this link for more information on FlexBox
